# AGENT REGISTRY AUDIT
## Anjusik Trading System


--------------------------------
PURPOSE
--------------------------------

Ensure the AGENT_REGISTRY.md accurately reflects the actual agent structure.


--------------------------------
AUDIT OBJECTIVES
--------------------------------

The audit verifies:

- registry matches agent prompt files
- no duplicate agents exist
- supervisor hierarchy is valid
- ownership authority remains intact


--------------------------------
AUDIT CHECKLIST
--------------------------------

1. Registry vs Files

Verify every agent listed in AGENT_REGISTRY.md has a corresponding prompt file.


2. File Location Check

Verify prompt files exist in correct department folders.


3. Duplicate Agent Check

Ensure no duplicate agent definitions exist.


4. Hierarchy Check

Verify supervisor relationships are valid.


5. Governance Protection

Verify no agent overrides owner authority.


--------------------------------
EXPECTED RESULT
--------------------------------

Each agent should have:

- registry entry
- prompt file
- defined supervisor
- defined restrictions


--------------------------------
AUDIT OUTPUT FORMAT
--------------------------------

Agent name  
Registry status  
Prompt file status  
Supervisor valid  
Notes


--------------------------------
AUDIT AUTHORITY
--------------------------------

Registry audits operate under the authority of ANJUSIK.
