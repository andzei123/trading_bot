# AI AUDIT PROCEDURE
## Anjusik Trading System


--------------------------------
PURPOSE
--------------------------------

This document defines the standard procedure for auditing the AI trading organization.

Audits verify:

- agent prompt integrity
- registry synchronization
- governance protection
- folder structure consistency
- escalation chain validity


--------------------------------
WHEN TO RUN AN AUDIT
--------------------------------

Audits should be executed when:

- new agents are added
- agent prompts are modified
- governance documents change
- major architectural updates occur
- before major system releases


--------------------------------
AUDIT STEP 1
--------------------------------
PROMPT INTEGRITY CHECK

Run PyCharm search checks:

AGENT NAME:
AGENT ROLE:
Supervisor:
RESTRICTIONS
OWNER AUTHORITY

Ensure prompt structure consistency.


--------------------------------
AUDIT STEP 2
--------------------------------
REGISTRY SYNCHRONIZATION

Verify that:

- every registry agent has a prompt file
- no duplicate agents exist
- supervisor chains match the registry


--------------------------------
AUDIT STEP 3
--------------------------------
FOLDER STRUCTURE CHECK

Verify that agents are located in correct folders:

leadership  
legal  
accounting  
intelligence  
strategy  
execution  
risk  
data  
monitoring  
research  
capital  
security  
support  


--------------------------------
AUDIT STEP 4
--------------------------------
GOVERNANCE SAFETY CHECK

Verify integrity of protected documents:

PROJECT_CONSTITUTION.md  
OWNER_PROTOCOL.txt  
AGENT_REGISTRY.md  
TREASURY_PROTOCOL.txt  

Ensure ownership attribution remains intact.


--------------------------------
AUDIT STEP 5
--------------------------------
INCIDENT RESPONSE VALIDATION

Confirm incident response chain exists:

Feed failure  
Runner crash  
Risk breach  
Delivery failure  
Governance violation


--------------------------------
AUDIT RESULT CLASSIFICATION
--------------------------------

GREEN

System architecture is valid.

YELLOW

Minor structural inconsistencies detected.

RED

Critical governance or ownership issue detected.
