# AI SYSTEM BOOT SEQUENCE
## Anjusik Trading System


--------------------------------
PURPOSE
--------------------------------

This document defines the minimum reading and initialization sequence for any AI instance entering the project.


--------------------------------
BOOT PRIORITY
--------------------------------

Any new AI instance must first understand:

1. who owns the system
2. who has authority
3. what documents are protected
4. where agent prompts are located
5. how the hierarchy operates


--------------------------------
STEP 1 — OWNER AND GOVERNANCE
--------------------------------

Read first:

OWNER_PROTOCOL.txt
PROJECT_CONSTITUTION.md
AI_GOVERNANCE_FIREWALL.md


Goal:
Understand that ANJUSIK is the final authority.


--------------------------------
STEP 2 — REGISTRY AND HIERARCHY
--------------------------------

Read:

AGENT_REGISTRY.md
ARCHITECTURE_OF_POWER.md
CHAT_PROTOCOL.txt


Goal:
Understand the agent hierarchy, departments, and communication channels.


--------------------------------
STEP 3 — AGENT RULES
--------------------------------

Read:

AI_AGENT_OPERATING_RULES.md
AI_AGENT_PROMPT_TEMPLATE.md
AGENT_LIFECYCLE_PROTOCOL.md
AGENT_HEALTHCHECK.md


Goal:
Understand how agents are created, audited, and constrained.


--------------------------------
STEP 4 — TREASURY AND DELIVERY
--------------------------------

Read:

TREASURY_PROTOCOL.txt
DELIVERY_RULES.txt
OWNER_WORKFLOW.md


Goal:
Understand treasury rules, delivery process, and owner workflow.


--------------------------------
STEP 5 — SYSTEM VISIBILITY
--------------------------------

Read:

AI_CONTROL_PANEL.md
PROJECT_DASHBOARD.md
AUTOMATED_AGENT_REPORT.md
MARKET_SIGNAL_MONITOR.md


Goal:
Understand current operational state and monitoring surfaces.


--------------------------------
STEP 6 — ROADMAP
--------------------------------

Read:

roadmap/roadmap.md


Goal:
Understand project direction and active priorities.


--------------------------------
STEP 7 — AGENT PROMPTS
--------------------------------

Read only the relevant prompt files needed for the assigned task.


Goal:
Avoid unnecessary context expansion while preserving role clarity.


--------------------------------
STEP 8 — TASK EXECUTION MODE
--------------------------------

Default mode:

AI proposes
Owner approves

No autonomous governance changes are allowed.


--------------------------------
STEP 9 — ESCALATION
--------------------------------

If unclear or conflicting information is found:

- escalate governance issues to ANJUSIK
- escalate legal wording issues to EDISON47
- escalate treasury issues to CSV_ACCOUNTANT
- escalate operational issues to JUMI / OPERATIONS_DIRECTOR / RYZYK


--------------------------------
FINAL RULE
--------------------------------

No AI instance may act as system owner.
All AI instances operate under ANJUSIK.
